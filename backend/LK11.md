## 1. Identifikasi Bug / Masalah

### Bug #1 — Sorting Frontend Hanya Berdasarkan Tanggal, Mengabaikan Waktu

**Lokasi:** `public/app/Pages/monitoring.php:16`, `pasien.php:228`, `detail-monitoring.php:30`

**Deskripsi:** Frontend melakukan `usort()` ulang terhadap data monitoring hanya menggunakan field `monitoring_date`. API Laravel sudah mengembalikan data terurut berdasarkan `monitoring_date DESC` dan `monitoring_time DESC`, tetapi setelah di-resort oleh frontend, data dengan tanggal yang sama kehilangan urutan waktunya. Contoh: kunjungan pagi (08:00) dan siang (14:00) pada hari yang sama bisa tampil terbalik.

**Dampak:** Ringan — urutan tampilan tidak akurat untuk pasien dengan multi-kunjungan dalam satu hari.

---

### Bug #2 — Hardcoded Fallback `Auth::id() ?? 1` di Admin PatientController

**Lokasi:** `app/Http/Controllers/Admin/PatientController.php:56`

**Deskripsi:** Saat membuat monitoring record bersamaan dengan pasien baru, terdapat fallback `Auth::id() ?? 1`. Jika autentikasi hilang di tengah request (misal session expired), monitoring akan tertaut ke user ID 1 tanpa verifikasi bahwa ID tersebut valid. Ini menciptakan phantom record yang bisa merusak integritas data.

**Dampak:** Sedang — potensi data monitoring terasosiasi ke user salah, sulit dilacak.

---

### Bug #3 — Missing Comma di Konfigurasi Sanctum Stateful Domains

**Lokasi:** `config/sanctum.php:22`

**Deskripsi:** Format string `sprintf('%s%s', ...)` tidak menyertakan koma antara daftar domain statis dan `Sanctum::currentApplicationUrlWithPort()`. Hasilnya domain aplikasi tergabung dengan `::1` menjadi `::1http://localhost:8000`, yang merupakan nilai malformed. Ini menyebabkan SPA stateful authentication gagal untuk domain aplikasi sendiri.

**Dampak:** Sedang — stateful SPA authentication tidak berfungsi optimal di production.

---

### Inconsistency (Terdeteksi, Tidak Diperbaiki sebagai Bug Terpisah)

usort($monitorings, fn($a,$b) =>
    strtotime($b['monitoring_date'] ?? '') <=> strtotime($a['monitoring_date'] ?? '')

usort($monitorings, fn($a,$b) =>
    strtotime(($b['monitoring_date'] ?? '') . ' ' . ($b['monitoring_time'] ?? '00:00:00')) <=>
    strtotime(($a['monitoring_date'] ?? '') . ' ' . ($a['monitoring_time'] ?? '00:00:00'))

- `public/app/Pages/pasien.php:228` — sortir monitoring terbaru per pasien
- `public/app/Pages/detail-monitoring.php:30` — sortir fallback monitoring berdasarkan pasien

### Bug #2: Hardcoded Fallback User ID

#### Sebelum (Admin/PatientController.php:56)
```php
'user_id' => Auth::id() ?? 1,
```

#### Sesudah
```php
'user_id' => Auth::id(),
```

**Catatan:** Route ini sudah dilindungi middleware `auth`, sehingga `Auth::id()` dijamin tidak `null`. Fallback ke `1` hanya menutupi error yang seharusnya terdeteksi.

### Bug #3: Missing Comma di Sanctum Config

#### Sebelum (config/sanctum.php:22)
```php
'stateful' => explode(',', env('SANCTUM_STATEFUL_DOMAINS', sprintf(
    '%s%s',  // ← tanpa koma
    'localhost,localhost:3000,127.0.0.1,127.0.0.1:8000,::1',
    Sanctum::currentApplicationUrlWithPort(),
))),
```

#### Sesudah
```php
'stateful' => explode(',', env('SANCTUM_STATEFUL_DOMAINS', sprintf(
    '%s,%s',  // ← koma ditambahkan
    'localhost,localhost:3000,127.0.0.1,127.0.0.1:8000,::1',
    Sanctum::currentApplicationUrlWithPort(),
))),
```

**Hasil:** Domain string sekarang berbentuk:
```
localhost,localhost:3000,127.0.0.1,127.0.0.1:8000,::1,http://localhost:8000
```
Parser `explode(',')` dapat memisahkan seluruh domain dengan benar.

## 3. Refactoring Kode

### Refactoring 1: Ekstraksi Fungsi `getStatusBadge()` ke Shared Helper

**Masalah:** Fungsi `getStatusBadge()` didefinisikan duplikat di 6 file berbeda dengan variasi minor:
| File | Baris | Variasi |
|------|-------|---------|
| `dashboard.php` | 35-44 | Tanpa emoji, label berbeda |
| `monitoring.php` | 34-41 | Standar ✅ / 🚨 / ⚠️ |
| `pasien.php` | 19-29 | Plus fallback dash "-" + match "control"/"kontrol" eksplisit |
| `jadwal.php` | 40-49 | Standar ✅ / 🚨 / ⚠️ |
| `hipertensi-diabetes.php` | 40-49 | Label "🚨 Rujukan" (tanpa "Perlu") |
| `lansia-pascarawat.php` | 40-49 | Label "🚨 Rujukan" (tanpa "Perlu") |

Inkonsistensi: `dashboard.php` tidak punya emoji, `hipertensi-diabetes.php` dan `lansia-pascarawat.php` pakai "🚨 Rujukan" sementara yang lain "🚨 Perlu Rujukan".

**Solusi:**

1. **Buat file** `public/app/Pages/helpers.php`:
```php
<?php

function getStatusBadge($status) {
    $s = strtolower($status ?? '');
    if (str_contains($s, 'stable') || str_contains($s, 'stabil')) {
        return '<span class="sv-badge sv-badge-stable">✅ Stabil</span>';
    }
    if (str_contains($s, 'referral') || str_contains($s, 'rujukan')) {
        return '<span class="sv-badge sv-badge-referral">🚨 Perlu Rujukan</span>';
    }
    return '<span class="sv-badge sv-badge-control">⚠️ Perlu Kontrol</span>';
}
```

2. **Include dari** `public/app/config.php` (line 7):
```php
require_once __DIR__ . '/Pages/helpers.php';
```

3. **Hapus definisi duplikat** dari:
   - `dashboard.php` (35-44)
   - `monitoring.php` (34-41)
   - `pasien.php` (19-29)
   - `jadwal.php` (40-49)
   - `hipertensi-diabetes.php` (40-49)
   - `lansia-pascarawat.php` (40-49)

**Keuntungan:**
- Semua 6 file menggunakan fungsi yang identik
- Label konsisten: ✅ Stabil / 🚨 Perlu Rujukan / ⚠️ Perlu Kontrol
- Perubahan di masa depan cukup edit 1 file

### Refactoring 2: Form Request Terpusat untuk Validasi Pasien

**Masalah:** Aturan validasi pasien didefinisikan terpisah di 2 controller dengan perbedaan:
- API Controller: `family_phone` punya `max:20`, `patient_category` punya `max:100`
- Admin Controller: tidak ada `max` constraint, plus ada field `monitoring_date`
- API menggunakan `Patient::create($validated)`, Admin mapping manual satu per satu

**Solusi:** Buat `App\Http\Requests\StorePatientRequest` yang menampung semua aturan validasi.

**File baru:** `app/Http/Requests/StorePatientRequest.php`
```php
<?php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;

class StorePatientRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true;
    }

    public function rules(): array
    {
        $patientId = $this->route('patient');

        return [
            'patient_id'       => $patientId
                ? 'sometimes|string|unique:patients,patient_id,' . $patientId . ',patient_id'
                : 'required|string|unique:patients,patient_id',
            'patient_name'     => 'required|string|max:255',
            'nik_dummy'        => 'required|string|max:20',
            'datebirth'        => 'required|date',
            'gender'           => 'required|in:Male,Female',
            'address'          => 'required|string',
            'family_phone'     => 'required|string|max:20',
            'patient_category' => 'required|string|max:100',
        ];
    }

    public function messages(): array
    {
        return [
            'patient_id.required'       => 'Kode pasien wajib diisi.',
            'patient_id.unique'         => 'Kode pasien sudah digunakan.',
            'patient_name.required'     => 'Nama pasien wajib diisi.',
            'nik_dummy.required'        => 'NIK wajib diisi.',
            'datebirth.required'        => 'Tanggal lahir wajib diisi.',
            'datebirth.date'            => 'Format tanggal lahir tidak valid.',
            'gender.required'           => 'Jenis kelamin wajib dipilih.',
            'gender.in'                 => 'Jenis kelamin harus Male atau Female.',
            'address.required'          => 'Alamat wajib diisi.',
            'family_phone.required'     => 'Nomor telepon wajib diisi.',
            'patient_category.required' => 'Kategori pasien wajib dipilih.',
        ];
    }
}
```

**Perubahan di** `Api\PatientController`:
```php
// Sebelum:
public function store(Request $request) {
    $validated = $request->validate([...]);
    $patient = Patient::create($validated);
}

// Sesudah:
public function store(StorePatientRequest $request) {
    $patient = Patient::create($request->validated());
}
```

**Perubahan di** `Admin\PatientController`:
```php
// Sebelum:
public function store(Request $request) {
    $validatedData = $request->validate([...]);
    $patient = Patient::create([...manual mapping...]);
}

// Sesudah:
public function store(StorePatientRequest $request) {
    $patient = Patient::create($request->validated());
}
```

**Keuntungan:**
- Single source of truth untuk validasi pasien
- Admin & API controller jadi lebih ramping (store dari 15+ line jadi 3 line)
- Mass assignment konsisten (`Patient::create($request->validated())`)
- Error messages dalam Bahasa Indonesia
